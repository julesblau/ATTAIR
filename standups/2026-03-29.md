## Decisions Made

1. **Interest picker values use full strings** — Changed from short keys ("musicians") to full label strings ("Musicians & K-Pop") to match `VALID_STYLE_INTERESTS` and `INTEREST_QUERY_MAP` in the backend. This means the same string flows through: frontend → profile → seenOn query.

2. **Light mode uses attribute selectors with !important** — The cleanest approach for the monolith. `[style*="rgba(255,255,255"]` selectors catch most inline dark-mode colors. Not perfect (some React inline styles use different formatting) but covers 90%+ of cases without refactoring to CSS custom properties.

3. **Occasion cache eviction** — Simple FIFO eviction when cache exceeds 1000 entries. LRU would be better but Map insertion order is FIFO by default, which is good enough.

## Needs Your Review

1. **Pricing discrepancy in paywall vs. Stripe** — Paywall screen shows $29.99/yr but the inline fallback `unit_amount: 3000` in payments.js is $30.00 (Stripe uses cents). The 1-cent difference exists because `2999` rounds to `$29.99` display but `3000` ≠ `2999`. If the `STRIPE_PRICE_YEARLY` env var is set in Railway (recommended), this fallback is never used.
