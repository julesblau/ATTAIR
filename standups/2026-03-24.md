## 🧠 Decisions Made

1. **Wishlists "add to list" via popover, not modal** — keeps it lightweight; a full modal felt like too much friction for a secondary action on a saved item card.

2. **Wishlist rename via contentEditable** — matches the existing scan rename pattern exactly; consistent UX.

3. **Paywall plan queued in sessionStorage** — so if a user selects "yearly" on the paywall before signing up, that plan carries through to checkout after account creation. Cleared immediately after use.

4. **Trial alert → in-app banner** — added `trialSuccess` state that triggers the same visual banner as upgrade success, auto-dismisses after 4s. Consistent with existing upgrade success flow.

5. **Pairings clicks tracked via existing `track()` helper** — not wired through full affiliate system since pairings don't have pre-existing click records in the DB. Revenue-optimal solution would be creating affiliate click records for pairings, but that's a separate backend change.

6. **`agents/` dir not committed** — Contains Playwright screenshots and test runner artifacts. Not meaningful as source files.


### 3. 🌐 SET VITE_SUPABASE_* IN VERCEL

- `VITE_SUPABASE_ANON_KEY=<new key after rotation>`

### 4. 📱 Consider removing phone number requirement from signup
E2E agent flagged: phone required = conversion killer. Instagram, Depop, and GOAT all make it optional. Backend already has it as required validation. Consider making it optional in both `auth.js` and the signup form.

### 5. 🎨 BANNER AD / SPONSORED placeholders
The ad banner and native ad slots show raw "BANNER AD" / "SPONSORED" text. This looks unfinished to beta users. Consider either a polished placeholder like the interstitial, or hide them entirely until real ads are integrated.

### 6. 🔗 Pairings affiliate tracking
Currently pairings clicks are tracked via analytics but not through the affiliate click system (no DB record, no redirect through `/api/go`). To earn commission on pairings clicks, the backend would need a new endpoint to create affiliate records for pairing suggestions. Flagging as a future revenue optimization.
